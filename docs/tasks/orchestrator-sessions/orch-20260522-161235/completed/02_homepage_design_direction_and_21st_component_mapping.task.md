## Objective
Define the final visual structure, component choices, and motion rules for the launch homepage.

## Scope
- map each homepage section to either custom implementation or a selected 21st.dev candidate
- define what stays from the current homepage and what is replaced
- define exact role of the hand/shield animation
- define doctor hero composition
- note performance and cohesion risks

## Context
The user wants strong personality, a doctor-led hero, and selective use of 21st.dev components. The hand/shield idea should survive, but not as a blocking intro.

## Definition Of Done
- section-by-section homepage recommendation exists
- 21st.dev candidates are mapped clearly
- hero, hand/shield, trust, services, doctor story, testimonials, CTA, and footer are all accounted for
- motion rules are explicit enough for a Gemini design handoff

## Expected Artifacts
- design recommendation document
- section/component mapping table
- notes on any components to avoid

## Constraints
- do not recommend using every flashy component
- prioritize performance, trust, and conversion
- keep the design suitable for a medical brand
